About Bitergia GitHub Development Analytics?

The Bitergia GitHub Development Analytics (AKA Cauldron for GitHub) is a proof of concept service designed by Bitergia for analyzing the activity in a GitHub repository.

It produces an snapshot report, as of the moment of the data retrieval, with metrics and charts about the activity in a repository, including git commits, issues and pull requests.

How the report is built?

Cauldron for GitHub is crafted from different Bitergia FOSS projects:

  1. It gathers data using some tools from the Metrics Grimoire toolkit:
    • CVSAnalY2 for getting git activity metadata.
    • Bicho using GitHub API for getting Issues and Pull Requests activity data.

      Note: GitHub API has some restrictions about the quantity of data it can gather (~ 5,000 requests per hour). If the repository has many issues or pull requests, maybe only a fraction of them will be retrieved for analysis.

  2. Information is stored in a SQL database.
  3. GrimoireLib produces a set of JSON files frorm the database.
  4. JSON files are visualized through Classic View (based on VizGrimoireJS) or Modern View, an specific visualization developed for this service: NG VizGrimoire for GitHub data.

Privacy

None user's private data is stored or published by Cauldron for GitHub. All the data used for the analysis are gathered from public repositories.
